# Lightweight 3D UAV Simulation

This is a browser-based Three.js simulation for demonstrating the project
story before a full ROS2/Gazebo runtime is available.

## Scope

The demo shows:

- indoor 3D flight volume;
- cylindrical/box obstacles and safety volumes;
- floor occupancy map generated from obstacle geometry;
- A*, RRT, and MPPI paths generated by the same Python grid-planner helper used
  by the ROS2 costmap bridge;
- UAV marker following the selected trajectory;
- compact metrics for path length, clearance, safety status, and 300-trial
  benchmark collision rate.

It is not a physics simulator and does not claim quadrotor dynamics.  It is a
visual simulation layer for explaining:

```text
obstacle representation -> occupancy/safety map -> planner path -> UAV motion
```

Gazebo/ROS2 runtime verification remains a separate step.

## Run

From the repository root:

```bash
scripts/serve_3d_simulation.sh 8765
```

Open:

```text
http://localhost:8765/
```

The server script refreshes:

```text
sim3d/uav_sim_data.json
```

before serving the page.

## Refresh Data Only

```bash
python3 scripts/export_3d_simulation_data.py --output sim3d/uav_sim_data.json
```

## Verify Rendering

The render verifier uses Playwright Core and a local Chrome/Chromium
executable.  It captures desktop and mobile screenshots and checks that the
WebGL canvas is nonblank.

```bash
NODE_PATH=/tmp/uav_oda_playwright/node_modules \
CHROME_PATH="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
node scripts/verify_3d_simulation_render.js http://localhost:8765/ outputs/figures
```

Expected outputs:

```text
outputs/figures/uav_3d_sim_desktop.png
outputs/figures/uav_3d_sim_mobile.png
outputs/figures/uav_3d_sim_render_check.json
```

## Record MP4

After starting the local server, record a deterministic 10-second A* demo:

```bash
NODE_PATH=/tmp/uav_oda_playwright/node_modules \
CHROME_PATH="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
node scripts/record_3d_simulation_video.js \
  --url http://localhost:8765/ \
  --planner astar \
  --duration-s 10 \
  --fps 24 \
  --width 1280 \
  --height 720 \
  --output outputs/videos/uav_3d_simulation_astar.mp4
```

Expected output:

```text
outputs/videos/uav_3d_simulation_astar.mp4
```

## Bundle

```bash
python3 scripts/bundle_3d_simulation_artifacts.py
```

Expected output:

```text
outputs/3d_simulation_artifacts.tar.gz
```

## Audit

```bash
python3 scripts/audit_3d_simulation_status.py --fail-on-incomplete
```

The audit checks scene JSON, required planner paths, desktop/mobile render
evidence, MP4 metadata, docs, and bundle presence.

## Use In Report/Presentation

Use this wording:

> A lightweight 3D simulation was added to visualize the current planner
> pipeline: obstacle geometry is converted into an occupancy/safety map, then
> A*/RRT/MPPI produce a path followed by a UAV marker.  This supports technical
> explanation and mentor demo.  Full Gazebo/ROS2 runtime evidence is still a
> separate verification step.
